Fake Telemetry is a simulation system demonstrating how software downloads and telemetry collection work in a controlled, transparent environment. It includes a fake software download website (HTML, CSS, JavaScript) styled like a real app page, linked to a Python telemetry agent executable. The backend server (Node.js, Express) receives telemetry data and logs it for analysis. The agent collects only non‑sensitive system information (OS, CPU, RAM, running processes) after explicit user consent, ensuring privacy. Data is transmitted securely to the local server (http://localhost:5000) and stored in log.txt. The project also integrates PyInstaller to convert the agent into a standalone executable, completing the end‑to‑end flow: website → download button → agent → backend → logs.
